## Section II: Prompt Engineering Masterclass & Context Management第二部分：快速工程大师班及情境管理

Garbage inputs (prompts) = garbage outputs (your answers) 垃圾输入（提示）= 垃圾输出（你的答案）

Bad prompting is the #1 most common mistake I see with any AI tool.糟糕的提示是我在任何人工智能工具中看到的最常见的错误。

It's in your best interests to learn prompt engineering - you'll save tokens (less cost/usage), and you'll save time having to re-prompt.学习提示工程对你大有裨益——你可以节省令牌（降低成本/使用量），还可以节省重新提示的时间。

Luckily, Anthropic has told us exactly how to prompt Claude for elite responses. 幸运的是，Anthropic 已经告诉我们如何才能让 Claude 给出精英级的回答。

You have two valid prompting structure options for Claude (beginner & advanced). 对于 Claude（初学者和高级玩家），您有两种有效的提示结构选项。

If you're a beginner, start here:如果你是新手， 请从这里开始：

**The 3-Part Prompt Formula三段式提示公式**

Every strong Claude prompt has three components. Stack them together, and your output goes from generic to genuinely useful.每个优秀的 Claude 提示都包含三个要素。将它们组合起来，你的输出就能从通用变为真正有用。

**1\. Set the stage1. 铺垫**

What's your role? What's the objective? Give Claude the context it needs before asking for anything.你的角色是什么？目标是什么？在向克劳德提出任何要求之前，请先提供必要的背景信息。

Example: "I'm building a website for a marketing landing page targeting Gen Z audiences."例如： “我正在为面向 Z 世代受众的营销落地页搭建网站 。”

**2\. Define the task2. 明确任务**

What specific action do you want Claude to take? Be direct and precise.你希望克劳德采取什么具体行动？请直接、准确地说明。

Example: "Write competitive copy, and build \[xyz\] sections." 例如： “撰写有竞争力的文案，并构建\[xyz\]部分。”

**3\. Specify the rules3. 明确规则**

Format, tone, length, style - tell Claude exactly how you want the output delivered.格式、语气、长度、风格——请告诉克劳德你希望以何种方式交付成果。

Example: "Keep it under 500 words."例如：“ 字数控制在 500 字以内。”

Build your prompts with these three components, and you'll get outputs better than 90% of people.用这三个要素构建你的提示，你的输出结果将优于 90% 的人。

3\. **Claude Skills** 3. **克劳德·斯基尔斯**

Found at **Main Homepage → Customize → Skills**.在**主页 → 自定义 → 技能**中找到 。

Think of Claude Skills as repeatable instructions & workflows for Claude.可以将 Claude 技能视为 Claude 的可重复指令和工作流程。

Think of Skills as repeatable instructions for Claude. Instead of typing the same prompt over and over, you turn it into a Skill, and Claude already knows exactly what to do.把技能想象成给克劳德的可重复指令。你不用一遍又一遍地输入相同的提示，而是把它变成一个技能，克劳德就知道该怎么做了。

Here's a real example: say you analyze spreadsheet data every day. Normally, you'd re-prompt Claude every single time: "Analyze this spreadsheet and look for XYZ." 举个实际例子：假设你每天都要分析电子表格数据。通常情况下，你每次都会重新提示克劳德：“分析这个电子表格，查找 XYZ。”

With a Skill, you'd just prompt "use my Spreadsheet Analyzer Skill," and it runs the same process automatically, every time, exactly how you want it. 有了技能，你只需提示“使用我的电子表格分析器技能”，它就会每次都自动运行相同的流程，而且完全按照你想要的方式进行。

The best part is, Claude can build these Skills for you - just ask it to build a Skill for \[insert workflow\]. 最棒的是，Claude 可以为你构建这些技能——只需让它为 \[插入工作流程\] 构建一个技能即可。

Multiple skills, combined into one automated function.多种技能融合到一个自动化功能中。

Instead of training Claude from scratch every time, the role is already defined.无需每次都从头开始培训克劳德，因为角色已经设定好了。
