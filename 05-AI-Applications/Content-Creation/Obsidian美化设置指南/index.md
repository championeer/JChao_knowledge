## Plugins插件

My ethos with plugins has always been to keep it quite simple. Hence I don’t use many plugins.我使用插件的理念一直是尽量保持简单。因此，我很少使用插件。

In all honesty, I found it a bit overwhelming the amount of different things that can be installed, and I was very hesitant to build workflows around plugins that were ultimately created by the community and there was no guarantee that they would be continued to be developed. I wanted to build a note-taking system that would last a long long time, and this was a brittleness I didn’t want to introduce.说实话，我发现可安装的插件种类繁多，有点让人应接不暇。我非常犹豫是否要围绕那些最终由社区创建的插件构建工作流程，因为无法保证它们会持续更新。我想要构建一个能够长期稳定运行的笔记系统，而这种脆弱性是我不想引入的。

I keep my usage to seven plugins, and I have been using these now for years. They are all installable via the community plugins finder, so I won’t include links to them specifically.我只用七个插件，而且已经用了好几年了。它们都可以通过社区插件查找器安装，所以我就不具体提供链接了。

**Calendar日历**

This works really well with the daily note core plugin - and is something I use pretty heavily. It shows a calendar in the sidebar, and you can easily transverse to a particularly daily note. It also visually shows you the days where you created a daily note with dots.它和 Daily Note Core 插件配合使用效果非常好——我本人也经常使用它。它会在侧边栏显示日历，方便你快速跳转到某条每日笔记。它还会用圆点直观地显示你创建每日笔记的日期。

I use this often when going back to the previous week to look for what I wrote on a specific day.我经常使用这个功能，以便回顾上周的内容，查找我在特定日期写的内容。

**Hider隐藏者**

Hider lets you hide/change parts of the Obsidian UI, and this is a critical plugin for getting the clean/minimalist aesthetic.Hider 插件可以隐藏/更改 Obsidian UI 的部分内容，对于实现简洁/极简主义的美学效果而言，这是一个至关重要的插件。

The settings I use are as follows:我使用的设置如下：

```text
Hide tab bar: Off
Hide status bar: On
Hide vault name: On
Hide scroll bars: On
Hide sidebar toggle buttons: Off
Hide tooltips: On
Hide file explorer buttons: On
Hide search suggestions: Off
Hide count of search term matches: Off
Hide instructions: On
Hide properties in reading view: Off
```

**Kanban看板**

I use the kanban plugin specifically to keep on top of writing tasks in my vault. This isn’t one I use extensively, but its definitely a nice quality of life improvement to be able to visualise the state of the different pieces of writing that sit within my vault.我专门使用看板插件来管理我的写作任务库中的写作任务。虽然我并不经常使用它，但它确实能很好地提升我的生活质量，让我可以直观地看到我任务库中不同写作任务的状态。

It lives within my “Outputs” folder.它位于我的“输出”文件夹中。

## Custom CSS自定义 CSS

I have only written custom CSS for one small aspect of my vault, which is the callouts.我只为我的保险库的一小部分（即标注）编写了自定义 CSS。

IMO callouts are a pretty underutilised feature in Obsidian generally, in that I rarely see others using them. If you have used Notion, you are likely familiar with the callout box, and that is something I use often there also.我认为 Obsidian 中的标注功能总体上被严重低估了，我很少看到其他人使用它。如果你用过 Notion，你可能对标注框很熟悉，我在 Notion 中也经常使用它。

Having a custom colour to match the monochrome nature of my vault was something I wanted to do - and it’s pretty easy in the sense you just create a .css file and then drop it in your ./obsidian/snippets folder. This can then be loaded under the “Appearance” tab in settings.我一直想为我的保险库设置一个自定义颜色，以匹配其单色调风格——这其实很简单，你只需要创建一个 .css 文件，然后将其放入 ./obsidian/snippets 文件夹即可。之后，就可以在设置的“外观”选项卡下加载该文件了。

The CSS for my callout snippet is:我的标注代码片段的 CSS 代码如下：

```css
.callout[data-callout=”north-star”] {
--callout-color: 158, 158, 158;
--callout-icon: sparkle;
}
```

This controls the colour, but it also controls the icon that appears at the top of the callout. I use the callout in my daily note for my daily goals, and have used a star icon.这不仅可以控制颜色，还可以控制标注顶部显示的图标。我会在每日笔记中使用标注来记录我的每日目标，并且一直使用星形图标。
