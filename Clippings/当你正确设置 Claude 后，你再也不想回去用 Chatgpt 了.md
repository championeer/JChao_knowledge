我每天接触的人，很多都在悄悄切换。 我关注的创作者、我服务的团队、私信里找我的创始人，一个接一个地不再打开 ChatGPT，最后都去了同一个地方：

Claude。克劳德。

我写 AI 已经三年了，发过上千条内容，累计阅读量几亿。所以经常有人问我：

“Ruben，我看到你写了很多关于某个 AI 工具的内容，但你自己真的在用吗？”

如果我写它，我就真的在用。

而在 2026 年 2 月这个时间点，Claude 是所有知识工作者最重要的 AI 工具。不是因为它完美无缺，它并不完美，我后面也会说它的短板。但它做得好的那部分，目前没有别的工具能真正接近。

这篇指南，就是我希望当初有人早点给我的那种指南。功能、安装、第一条 prompt，全部讲清楚。

建议你收藏下来，周末花 30 分钟把 Claude 真正用明白。

也转给那些总在问你： “老听人说 Claude，但我一直没试过”的人。

Claude 不是一个工具，而是六个工具。

如果你还以为 Claude 只是“Anthropic 版 ChatGPT”，一个聊天机器人，一个文本框，你输入一句，它回复一句，那是 2024 年的 Claude。

2026 年的 Claude，其实是六样东西：

- Cowork：一个能直接处理你真实文件的桌面应用
- Model：大多数人其实选错了 Claude 模型
- Excel：把 AI 装进你的电子表格
- Plugins：把 Claude 变成你这个职业的专家
- Artifacts：给你可直接使用的交互式输出，而不只是文字
- Projects：能记住所有内容的持久化上下文文件夹

下面我按重要性从高到低来讲。

1. Claude Cowork克劳德·科沃克

10 个字解释它是什么： 有点像 ChatGPT，但强很多。

为什么重要： Claude Cowork 直接运行在你的电脑上。它能读取你的文件，创建文档，生成表格，写你看不见但能帮你完成工作的代码。它在需要澄清的时候会主动问你问题，而不是瞎猜。

对知识工作者来说，Cowork 就像是 Claude Code 的对应版本。是的，Claude Code 不在这篇文章里，因为我的读者群体，包括我自己，并不写代码。但它一样优秀。

如何安装 Cowork： 去 [claude.com/download](https://claude.com/download) 下载应用。

你需要 Pro 账户，每月 20 美元；如果按年付，大约每月 17 美元。

打开应用，点击顶部的 Cowork 标签页。

从你的电脑里选一个文件夹，这样 Claude 就能读取里面的文件。

一个实用建议： 给你自己，或者任何你想让 Claude 了解的东西，单独建一些 markdown 文件。

我之前还写过一篇完整的 Cowork 使用指南。

你的第一条 prompt： 我想要 \[你的任务\]，这样成功的结果会是 \[你理想中的结果\]。

请先完整阅读我上传的文件，再开始回应。

先不要立刻执行。先通过 AskUserQuestion 向我提澄清问题，我们一步一步把方案打磨清楚。

只有在我们对齐之后，再开始动手。

关键点在于： 你要强迫 Cowork 先问问题。

它会开始生成一个表单式的提问流程，从而帮你拿到更好的答案。

我特别喜欢这个功能。甚至可以说，我现在都不需要一开始就把需求表达得很清楚了。Claude 会反过来逼我把事情说清楚。如果我觉得方向不对，我就直接告诉它，它会重新生成一套表单，在前面出错的基础上修正。

而且它有超过 100 万 token 的上下文窗口，也就是能基于海量对话内容进行推理，所以我几乎没感觉到 Cowork 在胡编乱造。

如果你是从 ChatGPT 迁移过来的，你要先完成一个思维转换：

ChatGPT 把你训练成了一个“更会写 prompt 的人”。 你写更长的 prompt，更巧的 prompt，甚至有一整个收藏夹放着各种模板 prompt，但你可能几周都没打开一次。

把这些先忘掉。

在 Claude Cowork 里，真正关键的是：文本文件。

把你知道的一切都写下来： 你的写作风格、品牌规则、最佳范例、过往作品，全都整理成 .md 或 .txt 文件，放进一个文件夹里，然后把 Claude 指向那个文件夹。

Claude 会先读这些文件，再开始回应。你给它的文件上下文越完整，你需要写的 prompt 就越少。

输出也会从“通用 AI 风格”，变成“这真的像我自己写的东西”。

这里还有个专业建议： 不要只是无脑上传几百份文本。要同时注意数量和质量。

一开始做这些文本文件确实要花点时间，但它的收益会不断复利，因为你以后会越来越少需要临时写 prompt。

我的做法是：

1. 认真写好 markdown 文件，就像给团队写 briefing 一样
2. 所有 prompt 开头都用一句固定话： “先读这些文件，然后问我问题，再帮我完成 \[任务\]。”

我基本上已经不再换别的 prompting 方式了。

我还专门写过一篇完整指南，讲怎么创建你自己的文本文件，从那里开始最好。

1. 选对 Claude 模型

正确答案是： Opus + Extended Thinking。正确答案是： Opus + 拓展思维。

现在你该选的模型是 Opus 4.6。 它在 2026 年 2 月 5 日发布，是目前最聪明的模型。无论是写作、思考、分析、规划，还是任何需要推理的任务，它都是最强的。

怎么设置： 打开任意一个 Claude 对话，无论是在 [claude.ai](https://claude.ai/) 还是 Cowork。

点击聊天窗口底部的模型选择下拉框。

选择 Opus 4.6 + Extended Thinking。选择 Opus 4.6 + 扩展思维。

步骤就是：

1. 点击模型
2. 选择 Opus 4.6
3. 打开 Extended Thinking开放延伸思维

一定不要忘了开 Extended Thinking。 它会强制 Claude 先“思考”再回答，有点像人在开口前的内心推演。差别非常大。

关于联网能力：

Claude 可以连接你的各种工具，比如 Slack、Google Drive、Notion、Figma，以及其他 50 多种服务。这些能力叫 Connectors。

去 Settings > Connectors。 浏览目录，点击 “Add” 就好了。

连上之后，Claude 就能在聊天过程中直接搜索你的 Slack 消息、读取 Google Docs、引用你的 Notion 页面。不需要复制粘贴，也不用发截图，它读取的是你真实的工作工具。

而且这项能力在所有套餐中都是免费的。

1. Claude in Excel克劳德在 Excel 中

10 个字解释它是什么： 一个住在你表格里的 AI，会读也会写公式。

为什么重要： 你一定试过把 Excel 文件上传给 ChatGPT。 然后它把所有内容压平成纯文本，公式没了，结构散了，还会一本正经地分析根本不存在的单元格。

Claude in Excel 不一样。 它直接活在你的电子表格里，能读取每一个工作表，也真的知道 D14 单元格里是什么。

安装方法，大约 3 分钟：

打开 Microsoft Excel，桌面版或者网页版都可以。需要 Excel 2016 或以上版本。

Windows 上进入 Insert > Get Add-ins， Mac 上进入 Tools > Add-ins。Windows 上进入插入>获取加载项， Mac 上进入工具>加载项。

搜索 Claude by Anthropic，认准官方版本和 Claude 标志。

点击 Add 或 Get It Now。

用你的 Claude 账户登录。

Mac 可以按 Ctrl+Option+C，或者直接在工具栏找到 Claude 图标。

你需要付费版 Claude 套餐，Pro、Max、Team 或 Enterprise 都行。插件本身是免费的。

你的第一条 prompt： 打开任意一个正在处理的电子表格，然后问：

“帮我总结每一个工作表的内容。”

接着再具体一点：

1. “用通俗语言解释一下 B12 单元格里的公式在做什么。”
2. “找出这个工作簿里所有 [#REF](https://x.com/search?q=%23REF&src=hashtag_click) 和 [#VALUE](https://x.com/search?q=%23VALUE&src=hashtag_click)[#REF](https://x.com/search?q=%23REF&src=hashtag_click) [＃价值](https://x.com/search?q=%23VALUE&src=hashtag_click) 错误。”
3. “把所有日期统一改成 YYYY-MM-DD 格式。”
4. “按产品类别生成每月收入的透视汇总。”

Claude 会高亮它改动过的每一个单元格。你能清楚看到改了什么，而且没有你的确认，它不会擅自执行。

如果你想从零开始做一个 Excel：

去 Claude Cowork，输入：

请为 \[用途\] 创建一个专业的 Excel 表格（.xlsx）。

背景：\[给谁用 / 怎么用\]

它应该包含：\[列出你要跟踪或计算的内容\]

规则：

- 使用 Excel 公式，如 SUM、SUMIF、IF 等，绝不要写死计算结果
- 所有可编辑的假设条件单独放在有标签的单元格中
- 冻结首行，自动调整列宽
- \[其他要求：图表、下拉框、条件格式、货币格式等\]

下面是一个示例 prompt：

请为一家 LinkedIn 代写 agency 制作一个专业的 3 年财务规划 Excel 文件（.xlsx），目标是在 3 年后以 1000 万美元退出。

背景： 这家 agency 的老板计划在 3 年内扩张并出售。潜在买家大概率会按收入倍数（2-4 倍）或 EBITDA 倍数（6-10 倍）估值。这个模型需要反推出达到 1000 万美元退出目标所需的增长路径。

应该包括：

- 退出情景分析：在不同倍数下，需要达到多少收入 / EBITDA 才能实现 1000 万美元退出
- 36 个月月度损益表：收入、COGS（写手/编辑成本）、运营费用、EBITDA
- 客户增长模型：客户数量、平均 retainer、流失率、每月新增客户
- 团队扩张模型：ghostwriters、editors、客户经理、每人成本
- 仪表盘：当前与目标的差距、所需月环比增长率、ARR 趋势图及不同退出场景对比

规则：

- 必须使用 Excel 公式，不允许硬编码计算结果
- 把可编辑假设条件单独放在有标签的单元格中
- 输入用蓝字，公式用黑字
- 货币格式为 $#,##0，百分比为 0.0%
- 冻结首行，自动调整列宽

而我从这个 prompt 得到的结果是： 这几乎是一键生成，而且效果惊艳，好到让我不敢相信现在已经这么容易了。

1. Claude 插件

10 个字解释它是什么： 预制好的技能包，让 Claude 立刻变专家。

为什么重要： Anthropic 在 2026 年 1 月发布了 11 个官方插件，覆盖销售、营销、法务、财务、数据分析、产品管理、客服支持等领域。

每个插件都给 Claude 注入了对应岗位的技能、工作流和斜杠命令。

比如你装了 Sales 插件，Claude 就能帮你做客户研究、电话前准备、外联文案、竞品 battlecard。 装了 Data 插件，它就能探索数据集、写 SQL、搭仪表盘、验证你的分析。

你不需要懂技术，只要点一下安装。

这些插件的重要性，甚至大到让一些法律软件公司在股市两天内蒸发了 2850 亿美元市值。Thomson Reuters 单日跌了 16%，创下历史最差单日表现，LegalZoom 跌了 20%。这不是一次小更新。

如何安装插件： 打开 Claude Cowork。

进入 [claude.com/plugins](https://claude.com/plugins)。进入 [claude.com/plugins](https://claude.com/plugins) 。

浏览插件列表，挑一个和你工作相关的。

点击安装，它会自动启用。

每个插件都会带自己的斜杠命令。

在聊天框里输入 / 就能看到有哪些命令可用。

你也可以直接问 Claude： “针对 \[你的工作职能\]，有哪些插件可用？”

安装插件后的第一条 prompt：

如果你安装的是 Marketing 插件：

/draft-content 写一篇关于 \[主题\] 的 LinkedIn 帖子。使用我上传的语气档案。目标读者是 \[受众\]。目标是 \[订阅转化 / 品牌曝光 / 互动\]。

如果你安装的是 Data 插件：

/build-dashboard 根据这个 CSV 创建一个交互式仪表盘。加上按日期和类别筛选的功能，并展示时间趋势。

1. Claude Artifacts克劳德文物

10 个字解释它是什么： Claude 里可直接用的交互输出，不只是文字。

怎么用： 在 Claude 和 Claude Cowork 里，Artifacts 会自动启用，不需要安装。

你的第一条 prompt： 创建一个交互式 HTML 计算器，把每月支出转换成全年预测。字段包括房租、食品、交通、订阅费用，以及一个会实时更新的总计。界面要简洁、极简。

你会看到什么？ Claude 直接在聊天里给你一个可运行的计算器。

还可以试这些：

1. “创建一个 \[产品 A\] 和 \[产品 B\] 的可视化对比图，设计简洁一点。”
2. “帮我做一个简单的项目跟踪器，包含任务、负责人、状态、截止日期四列。”
3. “用 SVG 画一个团队汇报关系图。”
4. Claude Projects克劳德项目

10 个字解释它是什么： 一组聊天和文件组成的文件夹，Claude 会记住你上传的一切。

怎么设置，大约 5 分钟：

登录 [claude.ai](https://claude.ai/)，需要 Pro 或 Team 计划。

点击左侧边栏的 Projects。

点击 Create Project，起个名字，比如 “My Newsletter”。

点击 Add content，上传你的关键文件：品牌文档、写作样本、参考资料、数据。别塞得太满。

说实话，我现在已经不太用 Projects 了。 它 bug 更多，而对我来说，Cowork + 本地 markdown 文件夹才是最好用的工作方式。

我也写过一篇文章专门讲我是怎么做这些文件的。

Claude 的短板在哪里

我说过会讲真话。

Claude 不会做图像生成。它不能直接生成照片、插画或视觉艺术作品。如果你需要这个，用 Gemini。

做图片的话，我现在推荐： Gemini + Nano-Banana 2 + Thinking（右下角打开）

做视频的话，可以用新的 Seedance 2.0，或者即将推出的 Gemini VEO-4。

Claude 也不是最强的实时搜索工具。它可以上网搜索，也能给你来自 Google 的信息，但在这一块，Grok 更强。我在别的文章里也解释过原因。

Grok 直接连着 X，而我 99.9% 的在线搜索需求都能在那里满足。

Claude 并不是每一项都最强，没有任何工具是。 但如果你的核心需求是写作、思考、分析、搭建内容、处理你自己的文件，那现在真的没有谁能打得过它。

你和 Claude 的前 30 分钟

打开日历，给自己和 Claude 预约 30 分钟。

第 0-5 分钟：安装 Claude 去 [claude.com/download](https://claude.com/download) 下载桌面应用 注册或登录 开 Pro，每月 20 美元 打开应用，点 Cowork

第 5-10 分钟：创建你的第一个文本文件 打开任意一个文本编辑器，比如 Google Docs。 新建一个文件，命名为 about-me。

写三件事：

1. 你是做什么工作的
2. 你喜欢怎样的沟通方式，比如正式、随意、直接
3. 一篇你自己最满意的写作样本

把它保存成 markdown 文件。

Markdown 是最适合大语言模型处理的文本格式之一。

一个小建议： 如果你不想打字，可以用 Wispr Flow，直接口述输入。

第 10-15 分钟：开始你的第一次 Cowork 对话 在 Claude Cowork 里，选择刚才那个文件所在的文件夹，也可以顺手再放几个文件进去。

输入： “先读 about-me 文件。然后基于它，帮我写 \[任务\]。”

看看会发生什么。 继续回应它，继续迭代。 对于那些你本来就很熟、但希望它帮你加速的任务，这种体验尤其惊艳。

第 15-20 分钟：试一个插件 在 Claude Cowork 里，点击聊天栏里的 “Plugins”。

浏览插件库，选一个适合你工作的。

适合入门的有：

- Productivity：任务和工作流
- Marketing：内容草拟
- Sales：客户研究

安装后，开一个启用插件的新对话。 输入 / 看看有哪些命令。 比如用 Marketing 插件时，你可以试：

/marketing:draft-post

然后描述你想写什么。

你会发现，输出比普通 prompt 更结构化，也更有判断力。

第 20-25 分钟：试一个 Artifact 在任意 Claude 对话中，让它做一个可视化的小工具。

比如输入： “创建一个交互式周计划模板，用 HTML 页面呈现，按周一到周五分列，时间从早上 9 点到晚上 6 点。”

和它互动，要求它修改，再把结果下载下来。

第 25-30 分钟：试试 Claude in Excel 打开 Excel。 进入 Insert > Get Add-ins。 搜索 Claude by Anthropic 并安装。

打开任意一个表格，问它： “解释一下这个带公式的单元格是什么意思。”

或者从零开始做一个新表格： 打开 Cowork 新对话 让它“从零创建一个 Excel” 给它出题 让它迭代 最后下载 .xlsx

真正该切换的原因

我不在乎 Claude、ChatGPT、Grok、Gemini，或者任何别的模型。

我不站队，也没有收钱写这篇 newsletter。

我只是每周两次分享，AI 正在如何非常快速地改变我的工作方式。因为我也在努力跟上，所以我希望你也能跟上。这样我们才能一起保持速度。

我想做的，是帮大家过滤掉 AI 世界里的噪音。 现在有 29 万多人每周两次读我的内容，关注的不是“有什么新模型”，而是“具体怎么用”。

有些人因为我的 LinkedIn 认识我。 但更多人订阅，是因为有人把我的文章转发给了他们。

如果这篇文章对你有帮助，那就把它转给另一个也需要的人。
