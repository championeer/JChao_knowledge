## WHY CURRENT PREDICTIONS ARE GHEY:为什么当前的预测不靠谱：

Simple: They treat humans like maths equations.很简单：他们把人当成数学方程式。

Give data.提供数据。

Run regression.运行回归分析。

Get probability.获取概率。

Ship data.船舶数据。

And that’s why it fails so frequently, these models collapse every time because we’re humans, not numbers, and we have emotions, feelings, get persuaded, and we’re influenced by our peers. This is what cascades through social networks, with information that mutates as it spreads, and creates outcomes that no spreadsheet could ever forecast.这就是为什么这些模型总是失效的原因，它们每次都会崩溃，因为我们是人，不是数字，我们有情感，有感觉，会被说服，也会受到同伴的影响。这就是社交网络中信息传播的机制，信息在传播过程中不断变异，最终产生任何电子表格都无法预测的结果。

Nothing is ever “rational”.世上没有什么是“理性”的。

## HOW A SIMULATION RUNS:模拟运行原理：

I know, I’m teasing you a little, but as soon as you read this MiroFish goes from “interesting concept” to “tool I need to deploy immediately.”我知道，我这是在逗你玩儿，但你读完这篇文章后，就会发现 MiroFish 从“有趣的概念”变成了“我需要立即部署的工具”。

STAGE 1: KNOWLEDGE GRAPH CONSTRUCTION第一阶段：知识图谱构建

Upload any text-based seed material (news articles, policy drafts, earnings reports, sentiment dumps, even fiction). Or just ask your prediction question in plain English.上传任何文本形式的种子材料（新闻文章、政策草案、盈利报告、情绪分析，甚至是小说）。或者直接用简单的英语提出你的预测问题。

GraphRAG then extracts every entity, maps every relationship, and identifies underlying power dynamics. The output becomes the bedrock reality of your simulated world.GraphRAG 随后提取每个实体，绘制每条关系图，并识别潜在的权力动态。输出结果构成模拟世界的基础现实。

STAGE 2: AGENT GENERATION第二阶段：代理生成

The engine populates your world with citizens.引擎会在你的世界中创造居民。

Each agent gets a distinct personality, background, occupation, socioeconomic status, and biases. University PR crisis as seed material? You get angry students, apathetic students, defensive faculty, concerned parents, local journalists. All with different baseline opinions.每个角色都有其独特的个性、背景、职业、社会经济地位和偏见。以大学公关危机为切入点？你会遇到愤怒的学生、冷漠的学生、充满防备的教职工、忧心忡忡的家长和当地记者。他们每个人都有着不同的基本观点。

An Environment Configuration Agent simultaneously sets the rules: information speed, exposure to opposing viewpoints, social network structure.环境配置代理同时设置规则：信息速度、接触对立观点、社交网络结构。

STAGE 3: PARALLEL SIMULATION第三阶段：并行仿真

OASIS drops agents into two parallel platforms running simultaneously:OASIS 将代理部署到两个同时运行的并行平台上：

They have their own social media’s with both short-form, emotional, high-virality interactions & longer-form, thread-based community debates.他们拥有自己的社交媒体平台，既有短篇幅、情感丰富、病毒式传播的互动内容，也有篇幅较长、以主题帖为基础的社区辩论。

Then you watch. Agents argue, persuade, form coalitions, break coalitions, radicalise, moderate. Exactly like real humans across real platforms.然后你就看着吧。经纪人们争论、劝说、结盟、瓦解联盟、激进化、温和化。就像真实平台上真实的人类一样。

The simulation runs in chronological “rounds” (40 recommended). Each round, every agent consults their Zep-powered long-term memory and decides how to react. Over those rounds, knee-jerk reactions evolve into echo chambers, polarisation, or consensus. Key Opinion Leaders emerge organically from the swarm.模拟以时间顺序进行，分为若干“回合”（建议 40 回合）。每一回合，每个个体都会查阅其由 Zep 驱动的长期记忆，并决定如何应对。经过这些回合，本能反应会演变为回音室效应、极化或共识。关键意见领袖（KOL）会从群体中自然涌现。

STAGE 4: REPORT GENERATION第四阶段：报告生成

A specialised ReportAgent audits everything. Opinion clusters, sentiment shifts across the timeline, narrative propagation pathways, most influential agents.专门的报告代理会对所有内容进行审核，包括观点集群、时间线上的情绪变化、叙事传播路径以及最具影响力的人物。

You find out why: Arguments are won. Coalitions are formed. You found out which: narratives gained traction and collapsed.你会发现原因：争论如何胜出，联盟如何形成。你会发现哪些叙事方式获得了支持，哪些叙事方式最终瓦解。

STAGE 5: THE “GOD’S-EYE VIEW”第五阶段：“上帝视角”

The simulation stays alive after the report. You get a dashboard where you can click on ANY agent and interrogate them directly.报告生成后，模拟过程仍会继续进行。您可以打开一个仪表盘，点击任何代理即可直接对其进行询问。

Why did they change their mind? Which narrative convinced them?他们为什么改变了主意？是哪种说法说服了他们？

But the real power is dynamic variable injection, because if you have a longer term prediction that’s been made and real world variables come into play, you can add them to see if that changes anything:但真正的强大之处在于动态变量注入，因为如果你已经做出了一个长期预测，而现实世界的变量开始发挥作用，你可以将它们添加进去，看看是否会改变什么：

## WHAT YOU NEED TO RUN IT:运行所需条件：

If you’re using cloud APIs (recommended for most people):如果您使用云 API（推荐大多数用户使用）：

Your local machine just acts as a director. The heavy cognitive processing happens on remote servers.你的本地计算机只是充当调度员的角色。繁重的认知处理工作是在远程服务器上进行的。

You need: 3 GHz CPU (2+ cores), 8 GB RAM minimum (16 GB recommended), and 32 Mb/s network bandwidth. A standard modern workstation handles this comfortably.你需要：3GHz 的 CPU（双核以上）、至少 8GB 内存（推荐 16GB）以及 32Mb/s 的网络带宽。一台标准的现代工作站就能轻松应对。

If you want to run it fully offline (MiroFish-Offline fork by nikmcfly):如果你想完全离线运行它（nikmcfly 的 MiroFish-Offline 分支）：

This community fork strips out all paid API dependencies. Translates the frontend from Chinese to English. Swaps Zep for a local Neo4j graph database. Runs LLMs locally through Ollama.这个社区分支移除了所有付费 API 依赖项，将前端从中文翻译成英文，用本地 Neo4j 图数据库替换了 Zep，并通过 Ollama 在本地运行 LLM。

But running massive language models locally demands serious hardware. The bottleneck is GPU VRAM. You need 10 GB minimum (for the 14-billion parameter model) or 24 GB recommended (for the 32-billion parameter model, typical of an RTX 3090 or 4090). 32 GB system RAM. NVMe SSD storage.但本地运行大型语言模型需要强大的硬件支持。瓶颈在于 GPU 显存。对于 140 亿参数的模型，至少需要 10GB 显存；对于 320 亿参数的模型（RTX 3090 或 4090 的典型参数量），建议使用 24GB 显存。此外，还需要 32GB 系统内存和 NVMe 固态硬盘。

CPU-only mode technically works but renders real-time simulation practically impossible. If you’re serious about local deployment, a dedicated high-VRAM GPU is non-negotiable.仅使用 CPU 模式虽然技术上可行，但实际上无法进行实时模拟。如果真要进行本地部署，那么配备独立高显存 GPU 是必不可少的。
