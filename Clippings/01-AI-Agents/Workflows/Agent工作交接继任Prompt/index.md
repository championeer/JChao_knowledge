使用 Codex / Claude Code / Gemini CLI 的一个技巧：如果对话太长，AI的幻觉越来越多，问题越来越难以解决，不妨开一个新的会话。在退出前让它做一下工作交接，Prompt 可以这样说：

> 本次对话的上下文已经太长了，我打算关掉并重新开一个新的会话。你有什么想对你的继任者说的，以便它能更好的理解你当前的工作并顺利继续？

最近我突然领悟到一项新技能，我称之为「Prompt 自举」，就是我先写一段 Prompt，算是我的原始需求，然后交给 AI 打磨，以实现「左脚踩右脚上天」的效果。

我知道这听起来好像没什么，但产生的效果惊人。我过去的很多所谓“自创”的 Prompt，在 AI 打磨后都变得惊人的好。

这是升级后的版本（建议无脑用这个版本就好。当然也可以根据自身需求，交给 AI 在此基础上继续迭代）：

> 19小时
> 
> 继任者 Prompt 2.0： 当前上下文已经过长，我将开启一个新的会话继续这项工作。 请你在 ./{yymmdd}-handoff.md 输出一份写给“下一位接手的 AI Agent”的交接摘要。假设它无法访问当前完整上下文，只能看到你现在这份交接内容，因此你的目标是让它尽快理解现状，并能直接继续推进工作。
